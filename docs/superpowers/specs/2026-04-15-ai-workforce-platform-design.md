# AI Workforce Platform — 设计文档

## 1. 产品定位

### 1.1 核心定位

AI员工管理与价值展示平台（MVP），将公司已落地的AI能力统一包装为"AI员工"形态，通过可视化看板和结构化管理界面，实现AI团队的监控、管理和价值展示。

### 1.2 目标受众

- **现阶段：** 公司高层（ROI验证、决策支持）+ 内部项目团队（日常监控管理）
- **后续阶段：** 外部投资人/客户（资本故事、产品展示）

### 1.3 业务背景

公司已构建24人AI员工团队，覆盖管理、设计、生产三条业务线，其中多个角色已在岗运行并产出可量化工作成果（如AI审计官累计846次审计、AI人力专员实现数据驱动人员盘点等）。现需将这些已运行的能力统一用平台化框架呈现。

### 1.4 数据现状

- 部分AI员工已落地运行，有真实生产数据，但缺少核心数据指标建设
- 部分AI员工在概念/开发阶段，先用 mock 数据把框架立起来
- 指标体系需从零设计，包含指标定义和计算逻辑

## 2. 技术方案

| 层面 | 选型 | 理由 |
|------|------|------|
| **前端框架** | Next.js (React) | 全栈一体、SSR/SSG支持、部署简单 |
| **UI** | shadcn/ui + Tailwind CSS | 深色科技风驾驶舱天然适配、组件质量高、高度可定制 |
| **数据可视化** | ECharts | 驾驶舱图表灵活度高、深色主题支持好 |
| **组织架构图** | React Flow | 节点拖拽、缩放、自定义节点样式，适合组织架构树 |
| **后端** | Next.js API Routes | MVP阶段前后端一体，轻量够用 |
| **数据库** | SQLite → PostgreSQL | 本地开发快，后续可平滑迁移 |
| **部署** | Vercel | 一键部署，给链接即可展示 |

## 3. AI员工价值度量指标体系

### 3.1 三层架构

```
平台级（驾驶舱首页）
  └── 团队级（按管理/设计/生产分组）
        └── 个人级（单个AI员工详情页）
```

### 3.2 平台级核心指标（驾驶舱展示）

| 指标 | 定义 | 计算逻辑 | 业务解释 |
|------|------|----------|----------|
| **AI员工总数/在岗数** | 团队规模与实际运行比例 | `在岗数 / 总数 × 100%` = 在岗率 | "我们有24个AI员工，19个已经在干活了" |
| **任务完成量** | AI员工执行并完成的任务总次数 | `SUM(已完成任务数)`，可切日/周/月 | "这个月AI团队一共完成了1284件事" |
| **等效人力节省** | AI完成同等工作对标人工所需时间 | `SUM(单任务人工基准耗时) - SUM(AI实际耗时)`，人工基准耗时可配置 | "AI团队这个月替代了47.5个人天的工作量，折合约14.25万元" |
| **平均采纳率** | AI产出被人类最终采纳使用的比例 | `被采纳的产出数 / 总产出数 × 100%` | "AI给出的建议/文档，有68.5%被人类真正用了" |
| **平均准确率** | AI产出经质检/人工校验的正确比例 | `通过校验数 / 总校验数 × 100%` | "AI做的东西，86.2%一次就过了质检" |
| **覆盖业务项目数** | AI参与的项目/流程总数 | `COUNT(DISTINCT AI参与的项目)` | "AI已经参与了423个业务项目" |

### 3.3 团队级差异化指标

#### AI管理团队

| 指标 | 计算逻辑 | 业务解释 |
|------|----------|----------|
| **审计覆盖率** | `已审计项目 / 应审计项目 × 100%` | "应该审的项目，AI审了多少" |
| **决策采纳率** | `执行的审计建议 / 总审计建议 × 100%` | "AI的审计建议，管理层真正执行了多少"（如AI审计官Q1采纳率61%） |
| **预警准确率** | `获明确管理结论的预警 / 总预警数 × 100%` | "AI发出的预警，有多少确实是需要处理的"（如AI人力专员85%获管理结论） |
| **闭环率** | `完成全流程闭环的事务 / 总事务数 × 100%` | "AI发起的事务，从头到尾走完整个流程的比例" |

#### AI设计师团队

| 指标 | 计算逻辑 | 业务解释 |
|------|----------|----------|
| **自动触发成功率** | `成功触发的事务 / 应触发事务 × 100%` | "该自动启动的任务，有多少真的启动了"（如AI产品经理的7类事务自动触发） |
| **文档采纳率** | `被用于下游的文档 / 总产出文档 × 100%` | "AI写的PRD/方案，有多少进了开发流程" |
| **阶段衔接自动化率** | `自动衔接的阶段 / 总阶段转换数 × 100%` | "项目从一个阶段到下一阶段，AI自动完成衔接的比例" |
| **方案修改轮次** | `AVG(人工修改次数)`，越低越好 | "AI出的方案，人要改几轮才能用——改得越少说明质量越高" |

#### AI生产团队

| 指标 | 计算逻辑 | 业务解释 |
|------|----------|----------|
| **产出量** | `SUM(资源产出件数)`，按类型分 | "AI生产了多少个剧本/角色/视频/音频等" |
| **入库成功率** | `成功入库数 / 总入库尝试数 × 100%` | "AI自动入库的资源，一次成功的比例" |
| **质检通过率** | `质检通过资源数 / 总质检数 × 100%` | "AI产出的资源，质检通过的比例"（如AI质检员打标准确率86%） |
| **平均生产耗时** | `AVG(任务完成时间 - 任务开始时间)` | "AI完成一个生产任务平均要多久" |

## 4. 功能模块设计

### 4.1 驾驶舱首页 `/dashboard`

**目标：** 一屏总览AI团队全貌，给管理者"掌控AI军团"的控制感。

**内容区块：**

1. **顶部核心指标卡片区** — 6大平台级指标（见3.2），每个卡片显示数值、环比趋势、简短解释
2. **三条业务线数据对比** — 柱状图/雷达图，横向对比管理/设计/生产团队的任务量、采纳率等
3. **AI员工活跃度热力图** — 24个员工的近30天活跃热力，快速识别谁在高频工作、谁处于空闲
4. **最近任务动态流** — 实时滚动的任务完成/启动事件流，营造"AI正在工作"的动态感

**交互：**
- 指标卡片可点击下钻到团队级看板
- 员工名称可点击跳转花名册详情
- 时间范围筛选器（本周/本月/本季度/自定义）

### 4.2 AI员工花名册 `/roster`

**目标：** 展示AI团队的完整阵容，支持按维度浏览和详情查看。

**列表页：**
- 顶部 Tab 切换：全部 / AI管理团队 / AI设计师团队 / AI生产团队
- 搜索栏 + 状态筛选（在岗/开发中/规划中）
- 卡片网格展示，每张卡片包含：
  - AI员工头像（生成式头像或自定义上传）
  - 名称、职位/Title
  - 状态徽标（绿色在岗 / 黄色开发中 / 灰色规划中）
  - 核心技能标签（最多展示3个）
  - 简要数据（本月任务数、采纳率/准确率）

**详情页 `/roster/:id`：**

4个 Tab 页签：

**Tab 1: Profile（档案）**
- 头像、名称、职位、所属团队
- Soul：AI员工的"灵魂"定义——核心使命和价值主张
- Identity：角色定位、性格特质、工作风格
- 个人简介：一段话描述这个AI员工是谁、做什么
- 所有字段支持手动编辑

**Tab 2: Skills（技能）**
- 技能列表，每个技能有：名称、描述、能力等级（1-5星或进度条）
- 擅长领域标签云
- 工具/平台接入情况（如对接了哪些系统）
- 支持手动添加/编辑/删除技能

**Tab 3: 数据（个人指标仪表盘）**
- 该员工的核心指标卡片（任务完成量、采纳率/准确率、人力节省）
- 趋势折线图（近6个月变化）
- 团队级差异化指标（根据所属团队展示对应的专属指标）

**Tab 4: 成长日志**
- 时间线形式展示版本迭代历程（V0.1 → V1.0 → V2.0...）
- 每个版本节点包含：版本号、日期、更新内容、能力变更说明
- 支持手动添加新的版本记录

### 4.3 生产看板 `/production`

**目标：** 实时监控AI团队的工作状态，查看历史产出。

**上半区 — 实时任务面板：**
- 卡片式展示当前正在执行的任务
- 每张任务卡片包含：
  - 执行者（AI员工头像+名称）
  - 任务名称、任务类型
  - 进度条 + 百分比
  - 开始时间、预估完成时间
  - 当前步骤说明
- 支持按团队筛选

**下半区 — 历史任务列表：**
- 表格形式，列：任务名称、执行者、类型、开始/完成时间、状态、耗时
- 支持搜索、筛选（按员工、按时间、按状态）
- 每行可展开查看产出内容摘要
- 点击进入任务详情页

**任务详情页 `/production/:taskId`：**
- 任务基本信息（名称、类型、执行者、时间）
- 执行步骤时间线（每个步骤的开始/完成时间、状态）
- 产出内容区（文档预览、资源预览、下载链接）
- 关联AI员工卡片

### 4.4 团队组织架构 `/org`

**目标：** 一图看懂24人AI团队的组织形态，用于对外展示时的"震撼一览"。

**展示形式：**
- 树状组织架构图，根节点为"AI Workforce"
- 第二层：三条业务线（管理团队10人 / 设计师团队4人 / 生产团队10人）
- 第三层：每个AI员工节点（头像+名称+状态）
- 节点可点击跳转到花名册详情页

**交互：**
- 支持缩放和拖拽
- 鼠标悬停节点显示简要信息（职位、核心技能、状态）
- 团队级节点显示该团队的汇总数据（在岗数、本月任务量）

### 4.5 系统设置 `/settings`

**功能：**
- **AI员工管理：** 新增/编辑/删除AI员工，批量导入
- **指标基准值配置：** 配置每类任务的人工基准耗时，用于计算"等效人力节省"
- **用户与权限：** 管理平台用户，区分管理员/普通用户角色（MVP阶段可简化）

## 5. 数据模型

### 5.1 核心实体

```
AIEmployee (AI员工)
├── id: string (UUID)
├── name: string (员工名称，如"AI审计官")
├── avatar: string (头像URL)
├── title: string (职位，如"项目审计专家")
├── team: enum (management | design | production)
├── status: enum (active | developing | planned)
├── soul: text (灵魂/使命定义)
├── identity: text (角色定位/性格特质)
├── description: text (个人简介)
├── createdAt: datetime
└── updatedAt: datetime

Skill (技能)
├── id: string
├── employeeId: string → AIEmployee.id
├── name: string (技能名称)
├── description: text (技能描述)
├── level: integer (1-5，能力等级)
└── category: string (技能分类)

Metric (指标数据)
├── id: string
├── employeeId: string → AIEmployee.id
├── period: string (如 "2026-03", "2026-W15", "2026-04-15")
├── periodType: enum (daily | weekly | monthly)
├── taskCount: integer (任务完成数)
├── adoptionRate: float (采纳率, 0-1)
├── accuracyRate: float (准确率, 0-1)
├── humanTimeSaved: float (节省人力，单位：小时)
├── customMetrics: json (团队专属指标，灵活扩展)
└── createdAt: datetime

Task (任务)
├── id: string
├── employeeId: string → AIEmployee.id
├── team: enum (management | design | production) (冗余字段，便于按团队筛选)
├── name: string (任务名称)
├── type: string (任务类型，如"项目审计"、"PRD编写"、"资源入库")
├── status: enum (running | completed | failed)
├── progress: integer (0-100)
├── currentStep: string (当前步骤说明)
├── startTime: datetime
├── estimatedEndTime: datetime
├── actualEndTime: datetime (nullable)
└── metadata: json (扩展信息)

TaskOutput (任务产出)
├── id: string
├── taskId: string → Task.id
├── type: enum (document | resource | report | media | other)
├── title: string
├── content: text (内容/摘要)
├── url: string (资源链接, nullable)
└── createdAt: datetime

VersionLog (成长日志)
├── id: string
├── employeeId: string → AIEmployee.id
├── version: string (如 "V1.0", "V2.0")
├── date: date
├── changelog: text (更新内容说明)
└── capabilities: json (能力变更详情)

MetricConfig (指标基准配置)
├── id: string
├── employeeId: string → AIEmployee.id (nullable, null表示全局默认)
├── taskType: string (任务类型)
├── humanBaseline: float (人工基准耗时，单位：小时)
├── costPerHour: float (人工时薪，单位：元，默认375元/天÷8h，用于折算节省金额)
├── description: string (配置说明)
└── updatedAt: datetime
```

### 5.2 关系

- AIEmployee 1 → N Skill
- AIEmployee 1 → N Metric
- AIEmployee 1 → N Task
- AIEmployee 1 → N VersionLog
- AIEmployee 1 → N MetricConfig
- Task 1 → N TaskOutput

## 6. UI 设计方向

- **整体基调：** 深色科技风（Dark Mode），呼应"AI/科技/未来"的产品调性
- **配色方案：** 深色背景（#0a0a1a ~ #1a1a2e）+ 高对比亮色数据色（蓝#00d4ff、绿#00ff88、红#ff6b6b、黄#ffd93d、紫#c084fc）
- **字体：** 中文用系统默认（苹方/微软雅黑），英文/数字用 Inter 或 JetBrains Mono（数据展示）
- **组件：** 基于 shadcn/ui 定制深色主题，卡片圆角、微光边框、渐变高亮
- **动效：** 数据数字跳动动画、卡片悬停浮起、图表加载动画——营造"活的系统"感觉
- **响应式：** 优先桌面端（大屏展示），适配平板（投资人用iPad看）

## 7. MVP 范围与边界

### 包含

- 驾驶舱首页（6大指标 + 图表 + 动态流）
- AI员工花名册（列表 + 详情4个Tab，支持编辑）
- 生产看板（实时任务 + 历史任务 + 产出查看）
- 团队组织架构图
- 系统设置（员工管理 + 指标配置）
- Mock 数据层（基于24个AI员工真实信息填充）
- 数据 API 接口（为后续接入真实数据源做准备）

### 不包含（后续迭代）

- 真实数据源对接（需要各AI系统提供数据接口）
- "找人办事"功能（直接在平台上给AI员工发起任务）
- AI员工之间的协作关系展示
- 多租户/多组织支持
- 移动端适配
- 用户认证/登录系统（MVP阶段可先不做，内网使用）

## 8. 项目分期建议

### Phase 1: 基础骨架（MVP核心）
- 项目初始化（Next.js + shadcn/ui + SQLite）
- 数据模型建表 + Mock 数据填充（24个AI员工完整信息）
- 全局导航 + 页面路由
- AI员工花名册（列表 + 详情页，含编辑功能）

### Phase 2: 驾驶舱与可视化
- 驾驶舱首页全部区块
- ECharts 图表集成（趋势图、对比图、热力图）
- 指标计算逻辑实现

### Phase 3: 生产看板与组织架构
- 生产看板（实时 + 历史 + 详情）
- 组织架构树状图
- 系统设置页面

### Phase 4: 打磨与部署
- UI 动效打磨
- 响应式适配
- Vercel 部署
- 性能优化
